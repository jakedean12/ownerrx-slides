Open Build Lab

Fix it, finish it, and get it ready to show.

You have 60 minutes. The demo is in 60 minutes. Don't plan — build. If something breaks, that's fine. If you can't fix it, skip it. Show what works.

  • Pairs work on their capstones independently.
  • Jake and any TAs circulate to unblock.
  • If the same blocker comes up twice, it gets addressed to the whole room.
  • Pairs with similar use cases can help each other.

Common Blockers — Fast Fixes

Blocker Fast Fix
Skill not firing Check the description field — does it match what you're saying? The model routes by description, not file name.
MCP not connecting Check auth token — regenerate if unsure. Most MCP auth failures are stale tokens.
Agent looping forever Tighten the goal in the system prompt. Open-ended goals cause open-ended loops.
Deployment 500 errors Check env vars — ANTHROPIC_API_KEY set? Most common cause of 500s.
Demo breaks on live data Have a fallback with hardcoded realistic data. Show it proudly — it proves you planned for fragility.
Antagonist too nice Add "Be adversarial. Find every flaw." to its system prompt. The default model is agreeable by nature.
Pipeline too slow Run independent stages in parallel. Sequential calls to the same model add up fast.
No eval cases Ask Claude Code: "Generate 10 realistic test cases for my capstone." Takes 30 seconds.
Part 2 · 7 min per pair

Demo Format

Every pair gets 7 minutes. Two valid formats. Both count equally.

Every pair gets 7 minutes. Hard stop. There are two valid formats — Shipped It and Killed It. Both count equally.

Format A
Shipped It
  1. 1 min: Who are you two, what company did you build for, what problem did you solve?
  2. 4 min: Live run — show real (or realistic) input, real output, one log/trace example, one eval case (golden or adversarial), and the human-approval gate.
  3. 2 min: What you'd build next + your handoff status. Both partners should speak.
Format B
Killed It (Equally Legitimate)
  1. 1 min: Who are you two, what was the original capstone, who was the sponsor?
  2. 4 min: What you built, what you learned, why it shouldn't ship. One partner covers the build, one covers the recommendation.
  3. 2 min: What the company should do instead — your written recommendation.
If your demo breaks live, walk through your incident response plan instead. That still counts. It shows you planned for failure.

Timing note: 7 minutes is strict. Both partners should speak during the demo. Rehearse to time — 7:20 is not 7:00.

Part 3

Graduation Checklist

A working demo alone does not graduate you. Every item below must be complete.

A working demo alone does not graduate you. Every item below must be complete before you call your capstone done.

Capstone Completion Checklist
  • Capstone pipeline running end-to-end — not "almost works," actually runs from input to output without manual intervention.
  • Eval set: 10–20 cases including at least 2 adversarial.
  • Structured logs + cost cap configured — every run produces a log, there is a ceiling on what it can spend.
  • PII redaction pass completed — or documented reason why not needed.
  • Deployment checklist completed and committed to the repo.
  • Post-deploy eval re-run: passing.
  • Security audit run with critical findings closed.
  • Handoff README: another teammate could follow it without asking you a single question.
  • Incident response plan written and committed.
  • Demo script rehearsed and timed at 7 minutes as a pair.

These aren't bureaucratic boxes — they are the actual difference between a project a company can run and a project that exists only on your laptop.
