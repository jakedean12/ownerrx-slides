Before You Install Anything

How KP Academy works

The whole program lives in one GitHub repo. Here's how the pieces fit together.

The repo is the course

There's no website, no video portal, no LMS. When you clone the kp-academy repo to your computer, you have the entire course. Every session guide, every homework skill, and your own personal workspace all live inside it.

Your participant folder: When you run the onboarding command, Claude creates a folder at participants/your-github-username/ that only you can write to. Everything you build across the program lives there — your notes, your agent code, your eval sets, your demo script.

Session slides

Before each session, you'll get a link to that session's guide (like this one). The guide walks you through what you'll do in class — concepts, exercises, and copy-paste prompts for Claude Code. You follow along during the live session.

Homework skills

After each session, you run a homework skill inside Claude Code. Type a command like /kp-session-1-homework and Claude reads your participant profile, then guides you through applied exercises tied to your specific capstone. No generic homework — everything is personalized to the company you're building for.

If a slash command doesn't fire: Type the natural phrase instead — start kp session 1 homework. This always works. Slash commands occasionally don't trigger; the phrase always does.

The Operator Track

Woven through every session is a consulting skills thread called the Operator Track. Each session teaches a technical skill and one of five consulting moves — defining a problem, designing an approach, communicating, showing results, tracking your work. By Session 8 you have both a technical portfolio and a consulting portfolio.

Step 2 — Mac

Install Claude Code on a Mac

Open Terminal (⌘ + Space, type "terminal") and run these commands one at a time.
Mac

1. Install Homebrew

Homebrew is the Mac package manager — it makes installing everything else one command. Paste this into Terminal:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

It asks for your Mac password. Let it run — takes 5–10 minutes.

2. Install Node.js, Git, and the GitHub CLI

brew install node git gh

3. Install Claude Code

npm install -g @anthropic-ai/claude-code
Permission error? Don't use sudo. Fix it with this instead: 
mkdir -p ~/.npm-global
npm config set prefix ~/.npm-global
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
Then retry the install command above.

4. Install a code editor (recommended)

Claude Code works in any terminal, but having a code editor open alongside helps you watch files change.

brew install --cask visual-studio-code

5. Verify everything installed

Run all four commands. Each should print a version number — not an error.

node --version
git --version
gh --version
claude --version
All four print numbers? You're ready. Move to Step 3 (connect Claude to Anthropic) below.

6. Sign in to Claude Code

You've been provided a Claude Code Pro account. Run claude and follow the browser login prompt — it will ask you to sign in with your claude.ai account.

7. Confirm Claude Code starts

claude

You should see Claude Code start up. Type exit to quit. If it starts — Mac setup is complete. Mac setup is complete.

Step 2 — Windows

Install Claude Code on Windows

Two paths. WSL is recommended — it gives you a real Linux environment that Claude Code expects.
Windows

Option A: WSL (recommended)

1
Install WSL. Open PowerShell as Administrator (right-click Start → Windows PowerShell (Admin)) and run: 
wsl --install
Restart your computer when asked. After reboot, Ubuntu will finish installing and prompt you to create a Linux username and password. Keep them simple — this is separate from your Windows login.
2
Open Ubuntu (search "Ubuntu" in Start menu) and run these commands: 
sudo apt update && sudo apt upgrade -y

curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install --lts

sudo apt install -y git

curl -fsSL https://cli.github.com/packages/githubcli-archive-keyring.gpg | sudo dd of=/usr/share/keyrings/githubcli-archive-keyring.gpg
sudo chmod go+r /usr/share/keyrings/githubcli-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/githubcli-archive-keyring.gpg] https://cli.github.com/packages stable main" | sudo tee /etc/apt/sources.list.d/github-cli.list > /dev/null
sudo apt update && sudo apt install -y gh

npm install -g @anthropic-ai/claude-code
3
Verify all four commands print version numbers: 
node --version
git --version
gh --version
claude --version

Option B: Native Windows (only if WSL won't install)

1
Install Node.js — download the LTS installer from nodejs.org, run with defaults.
2
Install Git — download from git-scm.com. During setup pick "Use Git from the command line" and "Checkout as-is, commit as-is".
3
Install GitHub CLI — open PowerShell and run: 
winget install --id GitHub.cli
4
Install Claude Code — open PowerShell (not Admin): 
npm install -g @anthropic-ai/claude-code
5
Verify: same four commands as above.
6
Install VS Code from code.visualstudio.com. If on WSL, also install the "WSL" extension.
