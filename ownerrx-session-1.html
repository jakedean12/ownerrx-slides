Who is running this

Alan Pentz founded OwnerRx to solve a specific problem: business owners know AI is important but don't know how to actually deploy it inside their companies. OwnerRx trains owners and their teams to build real agents for real workflows — not demos, not tutorials, things that actually run. Part 1

The agent loop

One concept before you touch anything — this is what Claude Code is actually doing.

Most people think Claude is a chat window. Claude Code is something different. It runs a loop:

Read files
→
Think
→
Run a command
→
Observe
→
Loop

It reads your files to understand context. It thinks about what to do. It runs a shell command or writes code. It observes what happened. Then it loops — reading again, re-evaluating, acting again — until it decides the task is done.

Why this matters: When you understand the loop, you stop being a passenger. You can see what Claude is about to do, approve or reject it, and step in when it goes off course. That's the skill this program teaches.

What this looks like in practice

You say: "Read the CSV in my folder and tell me which rows are missing a value."

Claude reads the CSV, figures out how to check for empty cells, runs a Python script, reads the output, then tells you which rows failed. You never typed a line of code. Your job is to give it good context and check what it's doing.

Part 2

Connect your GitHub account

Authenticate the GitHub CLI so your terminal can talk to GitHub directly.

Run this in your terminal

gh auth login
1
Where do you use GitHub? — select GitHub.com
2
How would you like to authenticate? — select Login with a web browser
3
It prints a one-time code. Copy it, press Enter — your browser opens GitHub
4
Paste the code on the GitHub page and authorize the app
5
Back in terminal you should see: Logged in as your-username

Verify it worked

gh auth status

You should see a green checkmark and your GitHub username. If you see an error, stop here and let your facilitator know.

What is gh? The GitHub CLI — it lets you clone repos, open pull requests, and check status directly from your terminal. You'll use it every session.
Part 3

Clone the course repo

Download a copy of the repo that stays connected to GitHub so you can push your work back up.

Clone and navigate into it

gh repo clone owner-rx/kp-academy
cd kp-academy

Confirm you're in the right place

pwd

You should see a path ending in /kp-academy. If it says /tmp/... or anything unexpected, stop and re-clone from your home folder.

See what's inside

ls

You'll see folders like participants/, content/, and docs/. Your work lives in participants/your-github-username/ — a folder only you can write to.

Important: Only write inside your own participants/ folder. The rest of the repo is CODEOWNERS-protected — Claude Code will warn you if you try to edit outside your folder.
Part 4

Open Claude Code and start your onboarding

Claude runs an intake interview and writes your profile to the repo.

Open Claude Code from inside the repo

claude

Claude Code opens. You'll see a prompt. Type exactly this:

start kp academy

Claude runs an onboarding interview. It asks about your background, your interests, and what you already know about AI. Answer in your own words — this profile is what every future session reads first so Claude already knows who you are.

What Claude writes for you

  • about.md — your background and profile. Every future session reads this first.
  • progress.md — your session tracker across the whole program.
  • artifacts/ and notes/ folders — where your work and notes will live.
  • Your first commit — pushed to GitHub with your name on it.
